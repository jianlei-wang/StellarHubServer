# TIF 转 COG 功能参数审查报告

本报告针对当前项目中 TIFF 转换为云优化 GeoTIFF (COG) 的流程进行了全面审查，详细罗列了涉及到的各项参数及其来源（外部输入或内部默认）。

## 1. 参数审查清单

| 参数名称 | 功能描述 | 来源 | 当前默认值 | 状态 |
| :--- | :--- | :--- | :--- | :--- |
| `input_path` | 输入 TIFF 文件的绝对路径 | 外部输入 | 无 | 已支持 |
| `output_path` | 输出 COG 文件的保存路径 | 外部/内部 | 自动生成 (原文件名 + `_cog.tif`) | 已支持自定义目录 |
| `dst_crs` | 目标坐标系 (重投影) | 外部输入 | 空 (不转换) | 已支持 |
| `profile` | COG 压缩配置文件 (Deflate, LZW等) | 外部输入 | `deflate` | 已支持可选 |
| `overview_level` | 金字塔 (Overviews) 的层级 | 外部输入 | `6` | 已支持可选 |
| `blocksize` | 瓦片 (Tile) 大小 | 外部输入 | `512` | 已支持可选 |
| `resampling` | 重投影时的重采样算法 | 外部输入 | `bilinear` | 已支持可选 (nearest, bilinear, cubic 等) |

## 2. 转换流程中的关键操作分析

### 2.1 坐标转换 (Reprojection)
- **当前逻辑**：如果 `dst_crs` 存在且与源不同，则先进行重投影，生成临时文件。
- **缺失灵活性**：重采样算法固定为 `bilinear`。对于离散型数据（如分类图、掩膜），`nearest` 更合适；对于平滑数据，`cubic` 可能更好。

### 2.2 金字塔构建 (Overviews)
- **当前逻辑**：固定设置 `overview_level=6`。
- **缺失灵活性**：小影像设置 6 层可能过多，超大影像设置 6 层可能不足。应允许用户干预或使用默认的 `auto` 模式。

### 2.3 COG 瓦片化 (Tiling)
- **当前逻辑**：使用 `rio_cogeo.cogeo.cog_translate`，未显式传递瓦片大小。
- **缺失灵活性**：默认瓦片大小通常为 512。某些特定场景（如特定前端加载优化）可能需要 256 或 1024。

## 3. 优化方案设计

为了提升工具的灵活性，计划在以下模块进行调整：

1.  **`cog_utils.py`**：修改 `tif_to_cog` 函数签名，增加 `**kwargs` 或显式参数以支持更多配置。
2.  **`cog_service.py`**：在业务层透传这些配置参数。
3.  **`cog_api.py`**：在 FastAPI 接口中增加可选的 Form 参数，并提供合理的默认值描述。
